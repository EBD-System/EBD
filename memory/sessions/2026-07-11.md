# Sessão 2026-07-11

## O que foi alterado
A edição de aluno foi alinhada com o fluxo manual validado no backend: a rota dedicada agora prioriza o nome atual do aluno como chave de edição e o cliente envia POST em formato `application/x-www-form-urlencoded` com a ação repetida na query string.

## Conhecimento consolidado
O backend aceita `updatealuno` em minúsculas e atualiza diretamente a aba `cadastro`. A edição deve usar o nome atual do aluno como identificador prático para evitar divergência com IDs derivados. O POST via Apps Script é mais confiável com corpo urlencoded do que com multipart para este fluxo.

## Próximos passos
Se houver novo erro de salvamento, verificar primeiro a ação recebida na query string, o deploy ativo do Apps Script e se a chave de edição que chega da UI continua sendo o nome atual do aluno.
