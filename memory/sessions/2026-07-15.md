# Sessão 2026-07-15

## O que foi alterado
- O cadastro público passou a usar `POST /auth/register` em JSON.
- A tela de login passou a usar `POST /auth/login` em JSON.
- A sessão autenticada agora preserva token e perfis retornados pelo backend.
- A rota `/login` ganhou entrada estática própria para funcionar em deploy estático.
- A tela de login ficou invisível quando a classe `access-self` ocultava `.app`; a regra agora excepciona o `data-route-mode="login"`.
- A tela de login foi simplificada para exibir apenas os campos necessários, com persistência opcional do nome de usuário e diálogo para "Esqueci minha senha".

## Conhecimento consolidado
- Cadastro e login oficiais são rotas JSON dedicadas em `/auth/*`.
- O frontend deve guardar a sessão retornada pelo backend no `localStorage` e pode guardar preferências de login, como o nome de usuário lembrado, no mesmo armazenamento.
- O cadastro público continua separado do shell autenticado.
